# 設定リロード機能

環境変数を再読み込みして、Botを再起動せずに設定を更新できる機能です。

## 機能概要

- 環境変数の再読み込み（`.env`ファイルから読み込み）
- 再読み込み後の設定情報の表示
- 管理者のみ実行可能

## コマンド

### `/config_reload`

環境変数を再読み込みします。

**権限**: 管理者のみ実行可能

**使用例:**
```
/config_reload
```

**実行後の表示:**
- リロード成功/失敗のメッセージ
- 再読み込みされた設定項目（Broadcast許可ユーザーなど）

## 使用シナリオ

1. `.env`ファイルの`BROADCAST_ALLOWED_USER_IDS`を変更した後
2. 他の環境変数を変更した後
3. 設定を更新したいが、Botを再起動したくない場合

## 注意事項

- 再読み込みされるのは`.env`ファイルから読み込まれる環境変数のみです
- システム環境変数は変更されません
- 一部の設定（Bot Tokenなど）は再読み込みしても反映されない場合があります（再起動が必要）
